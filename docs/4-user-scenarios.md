## 1. 개요

### 1.1 시나리오 문서란?

사용자 시나리오는 실제 사용자가 제품을 사용하는 구체적인 상황을 단계별로 기술한 문서입니다. 이를 통해:

- 개발자는 사용자 관점에서 기능을 이해
- 디자이너는 UX 흐름을 최적화
- QA는 테스트 케이스를 도출
- PM은 제품 방향성을 검증

### 1.2 시나리오 구조

각 시나리오는 다음 요소를 포함합니다:

- **시나리오 ID**: 고유 식별자 (예: SC-001)
- **페르소나**: 해당 시나리오의 주인공
- **목표**: 사용자가 달성하려는 것
- **전제 조건**: 시나리오 시작 전 상태
- **단계**: 사용자의 행동과 시스템 반응
- **결과**: 시나리오 완료 후 상태
- **감정**: 사용자의 감정 변화

|------|-------------|-------------|------|
| 1 | 사용자명: "ab" (3자 미만) 입력 | 실시간 검증: "사용자명은 3자 이상이어야 합니다" 에러 메시지 (빨간색) | ❌ |
| 2 | 사용자명: "valid_user" (올바름) | 에러 메시지 사라짐, 초록색 체크 표시 | ✅ |
| 3 | 이메일: "invalid-email" (@ 없음) | "올바른 이메일 형식이 아닙니다" 에러 | ❌ |
| 4 | 이메일: "user@example.com" | 에러 사라짐 | ✅ |
| 5 | 비밀번호: "1234" (8자 미만) | "비밀번호는 최소 8자 이상이어야 합니다" | ❌ |
| 6 | 비밀번호: "Pass1234!" | 에러 사라짐 | ✅ |
| 7 | "회원가입" 버튼 클릭 | 모든 검증 통과 → 계정 생성 성공 → 자동 로그인 | ✅ |

**교훈**: 사용자는 명확한 실시간 피드백을 받아 올바른 입력을 할 수 있음

3. API 요청 → 서버에서 JWT 검증 실패

   - 자동으로 로그인 페이지로 리다이렉트
   - 메시지: "로그인 세션이 만료되었습니다. 다시 로그인해주세요."
5. 사용자가 재로그인
6. 이전에 입력하던 작업은 유지되지 않음 (MVP)

**개선 방향 (Phase 2)**:
- 작업 중인 데이터를 LocalStorage에 임시 저장
- 재로그인 후 복구 제안

2. 제목 수정 중...

**[스마트폰]** (동시)

2. 마감일 변경: 2025-12-01 → 2025-12-05

**[데스크톱]** (계속)
