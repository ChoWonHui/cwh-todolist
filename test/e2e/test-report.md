### ✅ SC-001: 회원가입 플로우

**목표**: 새로운 사용자가 서비스에 회원가입하고 자동 로그인

**테스트 단계**:
1. ✅ 로그인 페이지 접속
2. ✅ "회원가입" 링크 클릭
3. ✅ 회원가입 폼 작성
   - 사용자명: `minji_kim`
   - 이메일: `minji@example.com`
   - 비밀번호: `MyPassword123!`
   - 비밀번호 확인: `MyPassword123!`
4. ✅ "회원가입" 버튼 클릭
5. ✅ 성공 알림 확인: "회원가입 성공!"
6. ✅ 자동 로그인 후 할일 목록 페이지로 리다이렉트

**결과**: ✅ **성공**

**스크린샷**:

### ✅ SC-002: 첫 할일 추가

**목표**: 사용자가 첫 번째 할일을 추가

**테스트 단계**:
1. ✅ 빈 할일 목록 화면 확인
2. ✅ "새 할일 추가" 버튼 클릭
3. ✅ 할일 정보 입력
   - 제목: "경영학 레포트 작성"
   - 설명: "마케팅 전략 사례 분석 5페이지"
   - 시작일: 2025-11-25
   - 마감일: 2025-12-02
4. ✅ "저장" 버튼 클릭
5. ✅ 할일 목록에 새 항목 표시 확인
   - 제목, 설명, 시작일, 마감일 정보 표시
   - 수정/삭제 버튼 표시

**결과**: ✅ **성공**

**스크린샷**:
- `04-todos-empty-page.png`: 빈 할일 목록
- `05-add-todo-modal.png`: 할일 추가 모달
- `06-add-todo-form-filled.png`: 할일 정보 입력 완료
- `07-todo-added.png`: 할일 추가 완료

### ✅ SC-005: 할일 수정 (마감일 연장)

**목표**: 기존 할일의 마감일을 수정

**테스트 단계**:
1. ✅ 할일 카드의 "수정" 버튼 클릭
2. ✅ 수정 모달에 기존 정보 표시 확인
3. ✅ 마감일 변경: 2025-12-02 → 2025-12-09
4. ✅ "저장" 버튼 클릭
5. ✅ 목록에서 변경된 마감일 확인

**결과**: ✅ **성공**

**스크린샷**:
- `08-update-todo-modal.png`: 할일 수정 모달
- `09-update-todo-date-changed.png`: 마감일 변경
- `10-todo-updated.png`: 수정 완료

### ✅ SC-004: 할일 삭제

**목표**: 완료한 할일을 삭제

**테스트 단계**:
1. ✅ 할일 카드의 "삭제" 버튼 클릭

   - 메시지: "경영학 레포트 작성 할일을 정말 삭제하시겠습니까?"

4. ✅ 할일이 목록에서 제거됨

**결과**: ✅ **성공**

**스크린샷**:

## 발견 사항

### 긍정적인 발견
1. ✅ **사용자 경험**: 모든 UI 요소가 직관적이고 사용하기 쉬움
2. ✅ **반응성**: 모든 작업이 즉시 반영되어 빠른 피드백 제공
3. ✅ **안전성**: 삭제 작업 시 확인 다이얼로그로 실수 방지
4. ✅ **데이터 검증**: 필수 필드(제목, 시작일, 마감일) 정상 작동
5. ✅ **자동 로그인**: 회원가입 후 자동 로그인으로 편의성 향상

3. 네트워크 오류 처리 - SC-019 시나리오 추가 테스트 필요
